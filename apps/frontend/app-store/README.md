# App Store materials — Eos

Drafts of everything App Store Connect asks for, in two grouped documents:

- [`metadata.md`](./metadata.md) — App Information + Version Information (name, subtitle, description, keywords, what's new, URLs, categories).
- [`review.md`](./review.md) — App Review Information (contact, demo account, notes for the reviewer) + age-rating answers.

## Screenshots

Generated by the `/appstore-prep` slash command into `apps/frontend/screenshots/<device>/` (gitignored). Re-run the command to regenerate; metadata above is not overwritten.

## Submitting

Manual: open each section, copy the block into the matching field in App Store Connect.

With Fastlane: this layout does **not** match the per-field `metadata/en-US/*.txt` files `fastlane deliver` expects out of the box. If you want to drive Fastlane, either (a) keep editing the markdown here and shell-script the split, or (b) re-introduce the per-field files alongside these docs. Most one-person submissions paste through the Connect UI and that's enough.
